# CC Switch 非官方修改版 v3.16.5-senvev47.1

本版本是基于 [farion1231/cc-switch](https://github.com/farion1231/cc-switch) 的非官方个人修改版，不是 CC Switch 官方发布版本，也不代表原项目维护者。

## 本次发布包含

- 新增图形化代理看门狗：在全局代理设置中管理上游代理地址、代理状态和检测结果。
- 支持三种代理模式：`自动`、`手动开启`、`手动关闭`。
- 手动优先级最高：处于 `手动开启` 或 `手动关闭` 时，不自动扫描、不自动切换。
- 自动模式下才会检测代理可用性，并根据结果在代理和直连之间切换。
- 新增当前路由状态显示：可看到当前是代理中还是直连中。
- 恢复 3.16.1 风格的 `测试模型` 按钮：它会发送一次最小化真实模型请求，并在供应商卡片上显示健康状态颜色。
- 保留 `检测连通` 作为轻量连通性检查，和 `测试模型` 的真实模型调用区分开。
- 调整内置更新入口，避免官方更新直接覆盖本 fork 的代理看门狗和测试模型改动。
- 增加手动 Windows x64 GitHub Actions 构建流程。

## 安装包

本 Release 提供 Windows x64 构建产物：

- `cc-switch.exe`
- `CC Switch_3.16.5_x64_en-US.msi`

发布包不包含用户本机的 CC Switch 数据库、日志、供应商配置、请求历史、API Key 或备份文件。

## 使用前请注意

- 这是非官方修改版。若你希望使用官方版本，请以原项目发布页为准。
- Windows 构建产物可能未签名，安装时请自行判断风险。
- `测试模型` 会向对应供应商发起一次真实请求，可能产生少量用量、费用、延迟或速率限制。
- 使用供应商、模型服务和代理服务时，请遵守对应服务条款及当地适用法律法规。
- 覆盖安装前建议自行备份 `~/.cc-switch/cc-switch.db`。

## 构建说明

本发布由 GitHub Actions 的手动 Windows 构建流程生成。构建流程会执行 TypeScript 类型检查并生成 Windows bundle。
