## 主要改动

### 1. 图形化代理看门狗

在全局出站代理设置中加入代理看门狗界面，用于在代理可用性变化时管理 CC Switch 的全局出站代理。

- 支持填写 HTTP/SOCKS 代理地址。
- 支持 `自动`、`手动开启`、`手动关闭` 三种模式。
- 手动优先级最高：处于 `手动开启` 或 `手动关闭` 时，不执行自动扫描和自动切换。
- 仅 `自动` 模式会定时检测代理可用性，并根据检测结果切换为代理或直连。
- 状态区会显示当前是否处在代理中、当前生效代理地址、最近一次检测结果。

### 2. 恢复 3.16.1 风格的测试模型按钮

供应商卡片中保留 `检测连通`，并恢复一个独立的 `测试模型` 操作。

- `检测连通`：轻量检查接口连通性，不发送真实模型对话请求。
- `测试模型`：调用后端 `model_test_provider`，发送一次最小化真实模型请求，用于验证供应商是否能实际完成模型调用。
- 测试结果会在供应商卡片上显示健康状态颜色，例如正常、降级或失败。

注意：`测试模型` 会向供应商发起真实请求，可能产生少量用量、费用或触发供应商侧速率限制。请只在你有权使用对应供应商与 API Key 的情况下运行。

### 3. 保护自定义功能不被官方更新覆盖

本 fork 调整了内置更新入口，避免直接通过官方更新覆盖本地修改功能。后续如需更新，建议从本 fork 同步上游代码后重新合并修改并构建。

### 4. GitHub Actions Windows 构建

仓库包含手动触发的 Windows x64 构建流程，用于在 GitHub Actions 中生成 `cc-switch.exe` 和 MSI 安装包。这样不需要在本机额外安装构建工具。

## 数据与隐私

本仓库和发布包不应包含任何用户本机的 CC Switch 数据库、日志、备份、API Key、供应商历史记录或请求历史。

不会发布的本机数据包括但不限于：

- `~/.cc-switch/cc-switch.db`
- `~/.cc-switch/logs/`
- 本地备份目录
- 用户自己的供应商 URL、Token、API Key、请求日志

升级或替换程序前，建议用户自行备份 `~/.cc-switch/cc-switch.db`。本 fork 的程序改动不需要清空数据库，也不会把用户历史数据打包进发布文件。

## 适合谁使用

这个修改版适合需要以下能力的用户：

- 希望在 CC Switch 图形界面里管理上游代理可用性。
- 希望代理自动切换只在自动模式下发生，并保留明确的手动优先级。
- 希望保留类似 3.16.1 版本中更直观的真实模型测试反馈。
- 希望通过 GitHub Actions 获取 Windows 构建产物，而不是在本机安装完整构建环境。
